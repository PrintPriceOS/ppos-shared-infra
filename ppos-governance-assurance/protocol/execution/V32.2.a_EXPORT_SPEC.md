# V32.2.a Backlog Export Spec v1.0.0

## 1. Objective
The **Backlog Export Spec (V32.2.a)** defines the technical schema for machine-readable representations of the PrintPrice OS Master Backlog. This enables automated integration with tools like Linear, Jira, and autonomous engineering agents.

## 2. Target Formats
- **Standard JSON (Hierarchical)**: Full program tree (Wave -> Epic -> Capability -> Story).
- **Tool Mapping (Linear/Jira)**: Flattened structure with specific metadata for external import.

## 3. Mandatory Metadata
Every story in the export MUST include:
- `id`: Unique technical ID (e.g., ST-101).
- `wave`: Implementation wave ID.
- `epic`: Parent technical epic.
- `capability`: Functional component.
- `repo`: Target physical repository.
- `acceptanceCriteria`: Array of verifiable strings.
- `dependencies`: Array of IDs that block this story.

## 4. Automation Policy
- Exports are generated deterministically from `master_backlog_catalog.json`.
- Mapping logic handles schema transformations (e.g., Label mapping for Linear).
