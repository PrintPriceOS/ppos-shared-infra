# V32.1.e Registry & Docs Spec v1.0.0

## 1. Overview
The **Registry & Docs Spec (V32.1.e)** defines the final wave of foundation repositories. These repositories serve as the "memory" (`shared-registry`) and "blueprint" (`docs-architecture`) of PrintPrice OS.

## 2. Target Repositories
- `ppos-shared-registry`: Centralized JSON registries for policies, capabilities, and configurations.
- `ppos-docs-architecture`: Canonical architectural documentation, specifications, and system diagrams.

## 3. Core Packages / Subdirectories
### `ppos-shared-registry`
- `policy-profiles`: JSON-based governance and trust policy definitions.
- `resource-catalogs`: Centralized lists of capabilities and organization types.

### `ppos-docs-architecture`
- `specs`: Formal service and protocol specifications.
- `milestones`: Roadmap and delivery milestone documentation.

## 4. Integration Strategy
- Registries are consumed as static data or through resolver services in other domains.
- Documentation serves as the ultimate source of truth for architectural drift detection.
- Unified root manifests maintained across all repositories.
