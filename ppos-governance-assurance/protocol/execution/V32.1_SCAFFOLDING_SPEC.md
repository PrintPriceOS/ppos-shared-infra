# V32.1 Repo Scaffolding Specification v1.0.0

## 1. Introduction
The **Repo Scaffolding Specification (V32.1)** defines the initial skeleton for all repositories. It ensures that every build starts with a consistent, contract-first structure aligned with the Master Implementation Blueprint (V32).

## 2. Repository Classes
- **Core Domain Repo**: (e.g., `ppos-core-platform`). Standard multi-package service repo.
- **Shared Contract Repo**: (`ppos-shared-contracts`). Authority for schemas and types.
- **Shared Registry Repo**: (`ppos-shared-registry`). Authority for JSON policies.
- **Docs Repo**: (`ppos-docs-architecture`). Centralized technical documentation.

## 3. Mandatory Root Manifest
Every repo MUST contain:
- `README.md`, `ARCHITECTURE.md`, `OWNERSHIP.md`, `CONTRIBUTING.md`.
- `package.json`, `pnpm-workspace.yaml`, `tsconfig.base.json`.
- `.gitignore`, `.eslintrc.cjs`, `.prettierrc`.

## 4. Package Structure
Canonical layout for `@ppos/` packages:
- `src/`: Services, Resolvers, Policies, Contracts, Schemas, Types.
- `tests/`: Unit, Integration, Robustness, Snapshots.
- `docs/`: README, CONTRACTS, TESTING.

## 5. Governance Rules
- No authoritative schemas in domain repos (duplicate block).
- All domain orchestrators must have a matching `.contract.` and `.schema.` pair.
- No repository creation without an assigned owning team (`OWNERSHIP.md`).
