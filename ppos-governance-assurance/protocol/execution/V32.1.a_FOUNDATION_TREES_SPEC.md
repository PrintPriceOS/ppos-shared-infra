# V32.1.a Foundation Repo Trees Spec v1.0.0

## 1. Introduction
The **Foundation Repo Trees Spec (V32.1.a)** provides the exact filesystem structures for the initial 6 repositories of PrintPrice OS. These trees are based on the **MVRS-1** (Minimum Viable Repo Scaffolding) set.

## 2. Foundation Set
1. `ppos-shared-contracts`: Schema authority.
2. `ppos-shared-registry`: Policy authority.
3. `ppos-docs-architecture`: Documentation authority.
4. `ppos-core-platform`: Kernel & Runtime.
5. `ppos-governance-assurance`: Governance & Trust.
6. `ppos-federation-protocol`: FEP & exchange protocol.

## 3. Global Structural Standards
- Every repo MUST have `packages/`, `docs/`, `tests/`, `scripts/`, `configs/`.
- Every repo MUST contain root manifests: `OWNERSHIP.md`, `ARCHITECTURE.md`, `CONTRIBUTING.md`.
- Every package MUST follow the `src/`, `tests/`, `docs/` internal layout.

## 4. Specific Trees (Excerpts)
Detailed trees for all 6 repositories are defined in the technical resolvers and registered in `foundation_repo_trees.json`.

## 5. Kickoff Order
1. Contracts -> 2. Registry -> 3. Docs -> 4. Platform -> 5. Governance -> 6. Federation.
